Algorithm Paradigms

Understanding the fundamental approaches to problem solving in computer science

Greedy Algorithms

A greedy algorithm builds a solution piece by piece, always choosing the next piece that offers the most immediate benefit. It makes the locally optimal choice at each step with the hope of finding a global optimum.

Key Characteristics
  • Fast execution - Typically O(n log n) time complexity
  • Simple implementation - Easy to understand and code
  • Low memory usage - No need to store subproblem solutions
  • No backtracking - Decisions are final once made
  • Not always optimal - May miss the globally best solution
When to Use

Use greedy when the problem has optimal substructure and the greedy choice property - meaning local optimal choices lead to global optimal solution.

Dynamic Programming

Dynamic Programming (DP) solves complex problems by breaking them into overlapping subproblems, solving each subproblem only once, and storing the results to avoid redundant computation (memoization).

Key Characteristics
  • Guarantees optimal solution - Explores all possibilities
  • Systematic approach - Builds solution from base cases
  • Avoids recomputation - Stores intermediate results
  • Higher complexity - Often O(n²) or O(n×W) time
  • More memory usage - Requires tables to store solutions
When to Use

Use DP when the problem has overlapping subproblems and optimal substructure, and when you need the guaranteed optimal solution regardless of computation cost.
